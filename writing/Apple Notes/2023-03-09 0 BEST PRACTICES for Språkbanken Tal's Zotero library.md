*0 BEST PRACTICES for Språkbanken Tal's Zotero library*
This entry updated **2023-01-17**.
This best practices document is a work in progress, and should be considered a starting point for citation management in the Språkbanken Tal's Zotero library. If you have any questions or concerns regarding the best practices outlined in this document, please bring them up during the weekly meetings.
**Please follow these instructions from top to bottom. It will save you time in the long run, and will ensure that the library is used effectively by all.**
1. **Plugins**  
	1. Install and use the following recommended add-ons. If you wish to use other add-ons, please discuss with the group first. You can find the recommended add-ons in \[the Zotero plugin listing\](https://www.zotero.org/support/plugins). The installation instructions may change, so be sure to check the link for the most up-to-date information.
	2. Keep Zotero and all installed plugins updated.
	3. **Required plugins**
	4. Better Bibtex (for locking bib keys, improved exports, and additional functionality.
	5. Zutilo (for linking to entries and collections, and additional functionality)
	6. ZotFile (for PDF management)
	7. Zotero Word and Libre Word integration (for seamless integration with word processing software)
	8. Zotero plugin for Chrome (this is installed from the Chrome web store)
1. Optional plugins
	1. Cita
	2. DOI manager (recommended for DOI management)
	3. sciteStorage scanner (for managing storage)
	4. Zotero Citation Counts Manager  
	5. Zotero PMCID fetcher (for fething PMCID)
1. **Adding new references**
	1. **Always see if it is already there first**
		1. Select the Språkbanken Tal root folder
		2. Type in (part of) the name of the publication in the search field at the top of the Zotero client
		3. If it is already there,just check that the entry meets the rest of the best practices (if not, fix it) and use the existing entry. You can pull it to a new sub-collection (this creates a link, it doesn't duplicate the entry) if it is useful.
		4. **NB!**  If you found the publication in the collection named “<a href="zotero://select/groups/2516652/collections/WJVUBK9E" rel="noopener" class="external-link" target="_blank"><b><u>Archive - don’t use as is</u></b></a>”, then
			1. **NB!**  Make sure you move the fixed entry to another suitable collection (**see 1 below**) outside of the archive
			2. **NB!**  Right/click on the netry *inside* the archive collection and select “Remove item from collection”
	1. **Find or create a suitable (sub)collection**
		1. Look through the tree of subocllecions to find a suitable subcolletion, or a suitable collection to hold a new subcollection if necessary. Typical collections would to use would be **Methodology**, **Papers** (for collections used in our own papers) **Speech technology basics** and **Speech science fundamentals**. Each *sub(collection should have, and many do have, a note with a title starting with **0. README**. Tis describes the collection, and also holds a pointer to SBTal best practices for Zotero.
		2. If this note isn#t present, please do add it as follows:
			1. Find such a note in another collection, preferably a parent of the one you are in.
			2. Right-click the note and chose “Duplicate”
			3. Select the new note, and change it to reflect the (sub)collection you’re adding a note to.
			4. Drag and drop the note to the collection you’re adding it to. As always, this will create a new reference (i.e. link) to the note.
			5. Check that the drag’n’drop succeeded.
			6. Go back to the collection where you duplicated the note and right click the edited duplicate. Chose “Remove from collection” (this removes the link in this collection, but leaves the note intact). **Do not chose “Move to bin”**, as this would remove the note everywhere.
	1. **Create the entry in the chosen (sub)collection**
		1. Although you can do this entirely by hand, the easiest way is to use the browser plugin. See to it that you stand in the chosen (sub)collection in the Zoterp program when you activate the plugin.
	1. **Edit the automatic citation**
		1. Zotero often get the publication type wrong. Check and correct. If you’re unsure what publication type to use, chances are you should use “Report”. Also, have a look at other publications in the SBTal library, but stick to those that are in a subcollection where the README says that the entries follow the SBTal best practices.
	1. **Lowercase title format**
		1. All titles should be in lowercase - no Title Case, no capitalisation except for the first letter and proper names. When you create reference lists for your article, some publishers will have other ideas about this. You'll have to change the output ***in the finished reference list*** if that is the case, not here.
	1. **Brief conference/workshop name in "Proceedings Title", full name in "Conference Name"** We're often short on space in reference lists (oddly enough), and in most cases the short proc. titles are a saviour. If you write for a publication requiring full proc titles, change the output *in the finished reference list* if that is the case, not here.
		1. Under "**Proceedings Title**", type e.g. "**Proc. of LREC 2018**"
	1. Under "**Conference Name**", type e.g. "**The Eleventh International Conference on Language Resources and Evaluation (LREC 2018)**" **If possible, add full author's names** There are too many Smith, A.'s out there. The software takes care of shortening for us in the output.
	2. **Download and add the full PDF and remove any other placeholders**
		1. If the auto-downloader cannot get the full text, download it manually and add it.
		2. Make an effort to find PDFs and digital books. The KTH library can often help. When you find them, add them. (It is very inefficient to search and potentially purchase publications more than once.)
	1. **Use the software to give PDFs and other media meaningful names**
		1. Once you've added a PDF, ebook or other media, check that the author, title, and date info for the entry are correct.
		2. Then right-click the media, select "*Rename File from Parent Metadata*".
		3. (You may have to close the file first, if you have it open in e.g. Acrobat Reader)
	1. **Once the reference is properly formatted, lock its Citation key**
		1. Select the reference.
		2. Right click.
		3. Select “Pin BibTex key” under “Better BibTex”.
	1. **Add the abstract**
		1. Most of the time the download browser plugin will manage this for you. Again, it is very helpful when using the refs to have the abstract accessible.
	1. **Leave country out for common locations** This is an old publication custom that we (and most if not all journals) adhere to.
		1. If the publication was published in either of Baltimore, Boston, Chicago, Los Angeles, New York, Philadelphia, San Francisco, Amsterdam, Jerusalem, London, Milan, Moscow, Paris, Rome, Stockholm, Tokyo, Vienna, leave the country out.
	1. **Add DOI as "Extra" for types with no DOI field**
		1. Zotero erroneously assumes that e.g. books do not have a DOI. If they do, add it under "**Extra**" as "**DOI: \[NUMBER\]**"
	1. **Use "Extra" field for structured key-value pairs (Metadata)**
		1. If you want to add some structured info, don't use a note, but use the extra field.
	1. **Use tags for labels, but have a look what tags are already in use first!**
		1. We try to use one tag only for each thing we want to label.
	1. **Use notes for unstructured info, e.g. narratives and free text.**

**Unsorted (fix!)**
1. Each reference has complete bibliographic info and that the title is cased in the European manner (no CamelCase, no Title-Case, just standard initial capital and capitals in acronyms and proper names)
2. Each reference has the article attached. If this is not possible, make a note explaining why.
